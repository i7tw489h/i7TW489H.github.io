Shell脚本循环的3个核心易错点

最近做Shell编程练习时，踩了不少循环相关的坑，整理了3个最容易出错的点，既是给自己复盘，也希望能帮到刚学Shell的同学～

一、变量一定要加双引号，避免空值报错

这是我踩得最多的坑！写until循环的终止条件时，没给变量加引号，导致变量为空时test命令直接报错。

错误写法

# 当$1为空时，这条命令会变成 test != ""，参数不完整，直接报错
until test $1 != ""
do
  echo "请输入有效参数"
  read input
  $1=$input
done
